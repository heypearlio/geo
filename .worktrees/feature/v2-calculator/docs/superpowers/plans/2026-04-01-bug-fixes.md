## Task 2: Fix score page graceful fallback when auditId is empty

**Root cause:** `app/score/page.tsx` line 492-493 immediately sets `auditState = "failed"` when `auditId` is empty. This happens because `submitAuditForm` in `actions.ts` returns `auditId: null` when the AuditSky API call fails or the key is missing. `audit/page.tsx` converts null to `""` with `?? ""`, producing `/score?auditId=&email=...`. The empty string is falsy, triggering the "failed" state instantly.

The quiz (QuizGame component) and the AISimulator already run without needing an auditId — only the score ring requires it. The fix: when auditId is empty, keep the quiz and interactive content visible, but replace the score ring section with a clear, warm message that drives the user to book a call instead of showing the "couldn't scan" error.
